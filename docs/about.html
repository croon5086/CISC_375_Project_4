Description of the tools (frameworks, APIs, etc.) used to create the application

  • Leaflet:
    The Leaflet API allowed us to render the interactive map of St. Paul. Also it allowed us to overlay data on top of the map (markers for the neighborhoods). The map was able to be modified by giving it coordinates and panning, and zooming.
  • Nominatim:
    The Nominatim API allowed us to search for locations by name, and also retrieve a location's name based on lat/long coordinates.
  • St. Paul Crime:
    We used our St. Paul Crime api to request JSON data for incidents and neighborhoods. Although we did not get the chance to fully implement a filtering feature for crimes and neighborhoods, our API can filter both
  • Bootstrap Framework:
    Bootstrap Framework was used to keep all of the content within the div tags properly sized based on the window.
  • Vue.js
    We stored the crime and neighbor hood data in an object that allowed us to dynamically display the data easier.

Six Interesting Findings

  1. In October 2019 the Most crime-infested neighborhood is Capitol River with a total of 909 incidents reported which is almost double that of the second most...the Payne/Phalen neighborhood with 473.
  2. Out of all the crimes committed in Capitol River in October 2019, a whopping 274 out of the 909 incidents were categorized as Proactive Police Visits.
  3. Proactive Police visits were the most common incident type across the city in October 2019.
  4. Least common reasons in October 2019 for Police Visits were: 1) arson 2) graffiti 3) Rape.
  5. The top three most common reasons in October 2019 for Police Visits were: 1) theft 2) vandalism 3)Narcotics.
  6. There is no recorded data in the database for any homicides in St. Paul in October 2019. (In reality, there were 3).
  7. In Union Park the most common incident in the whole database is theft. (St. Thomas is in Union Park and also Macalester-Groveland).
